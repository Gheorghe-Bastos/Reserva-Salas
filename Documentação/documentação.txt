Plano de Desenvolvimento — Reserva de Salas
Stack

Tech	Especificação

TypeScript	Obrigatório em todo o projeto
React	Vite (já configurado no projeto)
Banco	PostgreSQL via Neon (já é o que o projeto usa)
API	Própria (Fastify/Next route handlers) — vira diferencial, mas essencial pra nota de arquitetura
UI	Livre (Radix, Tailwind, shadcn/ui, MUI...)
Decisões a documentar (definir antes de codar)

Decisão	Sugestão

Reservas que encostam (termina 14h, outra 14h)	Não são conflito (horário final pertence à anterior)
Horário de funcionamento	Sim — 08h–18h, seg–sex (avisar se tentar fora)
Editar reserva com conflito	Bloquear igual criação, com mensagem clara
Comunicação de conflito no front	Modal/toast com: "Sala X já reservada das Y às Z"
Etapas do Desenvolvimento


Fase 1 — Base (BD + API + Schemas)

Tarefa	Detalhes

1.1 Modelar schema BD	Tabelas: rooms (id, nome, capacidade, created_at) e reservations (id, room_id, participante, inicio, fim, created_at). FK room_id → rooms, constraint CHECK (fim > inicio)
1.2 Migrations	Criar migration SQL e rodar no Neon
1.3 Tipos TypeScript	Interfaces Room, Reservation, CreateRoomDTO, CreateReservationDTO (compartilhadas entre front e API)

Fase 2 — Front-end Essencial

Tarefa	Detalhes

2.2 CRUD Reservas (UI)	Formulário com campos: sala (select), participantes, início, fim. Validação cliente (fim > início, campos obrigatórios)
2.3 Listagem com filtro	Filtro por sala (dropdown), ordenação por horário
2.4 Estados derivados	Abas/seções: Em andamento agora (início <= now < fim), Próximas (início > now), Encerradas (fim <= now)
2.5 Feedback de conflito	Exibir erro claro vindo da API: "Sala X já reservada das HH:MM às HH:MM"
2.6 Loading / vazio / erro	Estados de carregamento (spinner/skeleton), lista vazia ("Nenhuma reserva encontrada"), erro (toast/mensagem)


Fase 3 — Qualidade & Entrega

Tarefa	Detalhes

3.1 README	Stack, setup passo a passo, premissas e decisões documentadas, resposta da pergunta de raciocínio (reservas recorrentes)
3.2 Commits no GitHub	Commits atômicos contando a história do desenvolvimento
3.3 Vídeo (até 5 min)	Demonstrar: criar reserva, filtro por sala, conflito sendo bloqueado
3.4 Deploy (opcional)	Vercel para front + API (se for Next.js) ou separado


Ordem recomendada

Fase 1 (BD + API + tipos)  ──►  Fase 2 (Front)  ──►  Fase 3 (README + GitHub + Vídeo)
